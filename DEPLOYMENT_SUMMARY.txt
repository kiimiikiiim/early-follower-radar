EARLY FOLLOWER RADAR - DEPLOYMENT READY

PROJECT STATUS: PRODUCTION-READY

DEPLOYMENT OVERVIEW

Frontend: Next.js + Tailwind CSS
  - Deployment: Vercel (recommended)
  - Build: npm run build
  - Framework: Next.js
  - Styling: Tailwind CSS
  - Environment: NEXT_PUBLIC_API_BASE_URL

Backend: Express + SQLite
  - Deployment: Railway or Render
  - Build: npm install
  - Start: npm start
  - Database: SQLite (local file)
  - Scheduler: Every 6 hours

QUICK DEPLOYMENT (30 MINUTES)

1. PREPARE (5 min)
   - Push code to GitHub
   - Create Vercel account
   - Create Railway/Render account

2. DEPLOY FRONTEND (5 min)
   - Go to Vercel Dashboard
   - Import GitHub repository
   - Set root directory: frontend
   - Click Deploy

3. DEPLOY BACKEND (10 min)
   - Go to Railway/Render Dashboard
   - Create new project
   - Set root directory: backend
   - Add environment variables
   - Click Deploy

4. CONNECT (10 min)
   - Get backend URL from Railway/Render
   - Add to Vercel env vars: NEXT_PUBLIC_API_BASE_URL
   - Redeploy frontend

5. TEST (5 min)
   - Visit frontend URL
   - Check dashboard loads
   - Verify API connection

ENVIRONMENT VARIABLES

FRONTEND (.env.local / Vercel)
  NEXT_PUBLIC_API_BASE_URL = https://your-backend-url.railway.app

BACKEND (.env / Railway/Render)
  NODE_ENV = production
  PORT = 3000

DEPLOYMENT STEPS

STEP 1: PREPARE GITHUB
  git init
  git add .
  git commit -m "Initial commit"
  git remote add origin https://github.com/YOUR_USERNAME/early-follower-radar
  git push -u origin main

STEP 2: DEPLOY FRONTEND (VERCEL)
  1. Go to https://vercel.com/dashboard
  2. Click "Add New Project"
  3. Select GitHub repository
  4. Configure:
     - Framework: Next.js
     - Root Directory: frontend
  5. Click "Deploy"

STEP 3: DEPLOY BACKEND (RAILWAY)
  1. Go to https://railway.app/dashboard
  2. Click "New Project"
  3. Select "Deploy from GitHub"
  4. Configure:
     - Root Directory: backend
     - Build: npm install
     - Start: npm start
  5. Add Environment Variables:
     - NODE_ENV = production
     - PORT = 3000
  6. Click "Deploy"

STEP 4: CONNECT FRONTEND TO BACKEND
  1. Get backend URL from Railway
  2. Go to Vercel project settings
  3. Add environment variable:
     NEXT_PUBLIC_API_BASE_URL = https://your-backend-url.railway.app
  4. Redeploy frontend

STEP 5: TEST
  Frontend: https://your-app.vercel.app
  Backend: https://your-backend.railway.app/ping

TROUBLESHOOTING

Frontend shows "Failed to connect":
  1. Check backend URL in Vercel env vars
  2. Verify backend is running
  3. Check CORS headers
  4. Redeploy frontend

Backend returns 404:
  1. Test health: curl https://your-backend-url/ping
  2. Check logs in Railway/Render dashboard
  3. Verify build completed successfully

No signals showing:
  1. Run scraper: node runScraper.js
  2. Check: curl https://your-backend-url/api/signals
  3. Verify database has data

CORS errors:
  1. Check backend CORS is enabled
  2. Verify frontend URL in CORS config
  3. Check browser console for actual error
