# Signal Endpoints Documentation

## Overview

The Express backend now includes three new endpoints for retrieving signal data from `signals.json`. All endpoints return JSON data with CORS enabled for frontend access.

## Base URL

```
http://localhost:4000
```

## Endpoints

### 1. GET /api/signals

Returns all signals from `signals.json`.

**Request:**
```bash
curl http://localhost:4000/api/signals
```

**Response:**
```json
{
  "success": true,
  "count": 6,
  "signals": [
    {
      "username": "tech_writer",
      "followed_by": ["TechCrunch", "VentureBeat", "forbes"],
      "score": 3
    },
    {
      "username": "ai_researcher",
      "followed_by": ["elonmusk", "sama"],
      "score": 2
    },
    ...
  ]
}
```

**Response Fields:**
- `success` (boolean): Whether the request was successful
- `count` (number): Total number of signals
- `signals` (array): Array of signal objects

**Signal Object:**
- `username` (string): The account handle
- `followed_by` (array): List of big accounts following this account
- `score` (number): Number of big accounts following (= length of `followed_by`)

**Status Codes:**
- `200`: Success
- `500`: Server error

---

### 2. GET /api/signals/high

Returns only high-signal accounts (score >= 2).

**Request:**
```bash
curl http://localhost:4000/api/signals/high
```

**Response:**
```json
{
  "success": true,
  "count": 5,
  "signals": [
    {
      "username": "tech_writer",
      "followed_by": ["TechCrunch", "VentureBeat", "forbes"],
      "score": 3
    },
    {
      "username": "ai_researcher",
      "followed_by": ["elonmusk", "sama"],
      "score": 2
    },
    ...
  ]
}
```

**Filtering:**
- Only returns accounts where `score >= 2`
- Sorted by score (highest first)

**Use Cases:**
- Dashboard "High Signal" section
- Alerts for high-signal accounts
- Priority screening

**Status Codes:**
- `200`: Success
- `500`: Server error

---

### 3. GET /api/signals/recent

Returns the latest 20 signals.

**Request:**
```bash
curl http://localhost:4000/api/signals/recent
```

**Response:**
```json
{
  "success": true,
  "count": 6,
  "signals": [
    {
      "username": "tech_writer",
      "followed_by": ["TechCrunch", "VentureBeat", "forbes"],
      "score": 3
    },
    {
      "username": "ai_researcher",
      "followed_by": ["elonmusk", "sama"],
      "score": 2
    },
    ...
  ]
}
```

**Behavior:**
- Returns first 20 signals from the sorted list
- Signals are pre-sorted by score (highest first)
- If fewer than 20 signals exist, returns all

**Use Cases:**
- Dashboard "Recent Follows" section
- Activity feed
- Latest discoveries

**Status Codes:**
- `200`: Success
- `500`: Server error

## CORS Configuration

All endpoints have CORS enabled. The backend accepts requests from any origin:

```javascript
app.use(cors());
```

**Allowed Methods:**
- GET

**Allowed Headers:**
- Content-Type
- Authorization

## Data Source

All signal endpoints read from:

```
backend/data/signals.json
```

**File Format:**
```json
[
  {
    "username": "account_handle",
    "followed_by": ["big_account1", "big_account2"],
    "score": 2
  }
]
```

The file is generated by `detectSignals.js` and updated automatically by `runScraper.js`.

## Examples

### JavaScript/Fetch

```javascript
// Get all signals
fetch('http://localhost:4000/api/signals')
  .then(res => res.json())
  .then(data => console.log(data.signals));

// Get high-signal accounts
fetch('http://localhost:4000/api/signals/high')
  .then(res => res.json())
  .then(data => console.log(data.signals));

// Get recent signals
fetch('http://localhost:4000/api/signals/recent')
  .then(res => res.json())
  .then(data => console.log(data.signals));
```

### React/Next.js

```javascript
import { useEffect, useState } from 'react';

export default function SignalsPage() {
  const [signals, setSignals] = useState([]);
  const [highSignals, setHighSignals] = useState([]);

  useEffect(() => {
    // Fetch all signals
    fetch('http://localhost:4000/api/signals')
      .then(res => res.json())
      .then(data => setSignals(data.signals));

    // Fetch high-signal accounts
    fetch('http://localhost:4000/api/signals/high')
      .then(res => res.json())
      .then(data => setHighSignals(data.signals));
  }, []);

  return (
    <div>
      <h2>All Signals ({signals.length})</h2>
      {signals.map(signal => (
        <div key={signal.username}>
          <p>{signal.username} - Score: {signal.score}</p>
        </div>
      ))}

      <h2>High Signal ({highSignals.length})</h2>
      {highSignals.map(signal => (
        <div key={signal.username}>
          <p>{signal.username} - Score: {signal.score}</p>
        </div>
      ))}
    </div>
  );
}
```

### cURL

```bash
# Get all signals
curl http://localhost:4000/api/signals | jq '.'

# Get high-signal accounts
curl http://localhost:4000/api/signals/high | jq '.signals'

# Get recent signals
curl http://localhost:4000/api/signals/recent | jq '.signals | length'

# Get specific account

# Get accounts followed by specific big account

```

## Error Handling

### Missing signals.json

If `signals.json` doesn't exist:

```json
{
  "success": true,
  "count": 0,
  "signals": []
}
```

### Invalid JSON

If `signals.json` contains invalid JSON:

```json
{
  "error": "Error loading signals: Unexpected token..."
}
```

**Solution:** Verify `signals.json` is valid JSON or regenerate it with `detectSignals.js`.

### Server Error

If an unexpected error occurs:

```json
{
  "error": "Internal server error"
}
```

**Status Code:** `500`

## Troubleshooting

### Endpoints Return Empty

**Problem:** All endpoints return `count: 0`

**Causes:**
- `signals.json` doesn't exist
- `signals.json` is empty
- Scraper hasn't run yet

**Solution:**
```bash
# Run the scraper
cd backend
node runScraper.js

# Verify signals.json was created
ls -l data/signals.json
cat data/signals.json
```

### CORS Errors in Frontend

**Problem:** Frontend gets CORS error

**Cause:** CORS not enabled or misconfigured

**Solution:** Verify CORS middleware in `index.js`:
```javascript
const cors = require('cors');
app.use(cors());
```

### Stale Data

**Problem:** Frontend shows old signals

**Cause:** Signals haven't been updated

**Solution:**
```bash
# Check when signals.json was last updated
stat data/signals.json

# Run scraper to update
node runScraper.js

# Verify new signals
cat data/signals.json
```

## Best Practices

1. **Cache Results**: Cache API responses in frontend for 30 seconds
2. **Error Handling**: Handle empty signals gracefully
3. **Loading States**: Show loading indicator while fetching
4. **Auto-Refresh**: Update signals periodically (30-60 seconds)
5. **Filtering**: Use `/api/signals/high` for important signals
6. **Pagination**: Use `/api/signals/recent` for activity feed
